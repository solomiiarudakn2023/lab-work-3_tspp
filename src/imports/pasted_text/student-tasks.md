ДОДАТКОВІ ЗАВДАННЯ ДЛЯ СТУДЕНТІВ (УМОВНО ОБОВ'ЯЗКОВІ)
Студенти можуть виконати 2 завдання (для тих, хто претендує на "4") або 4 завдання (для тих,
хто претендує на "5"), щоб отримати бонусні бали. Ці завдання дозволять розширити
функціональність проєкту, додати інтерактивність і покращити дизайн, використовуючи
сучасні технології frontend.
1. Анімація інтерфейсу
Додати анімацію переходів між сторінками або анімацію елементів (наприклад, плавна
поява тексту, ефекти наведення на кнопки, завантаження контенту).
Використати CSS-анімації (@keyframes, transition, transform) або JavaScript (GSAP,
Intersection Observer).
Приклад: в інтернет-магазині зробити анімацію додавання товару в кошик.
2. Темна та світла тема
Реалізувати перемикач тем (dark/light mode) для сайту.
Використати CSS-змінні або JavaScript для зміни теми.
Додати збереження вибраної теми в localStorage.
Приклад: особистий блог або музичний сайт, де користувач може перемикати тему
дизайну.
4. Додавання форми з валідацією
Додати форму (реєстрації, зворотного зв’язку, запису на прийом) із валідацією полів.
Використати HTML5-атрибути (required, pattern), JavaScript або бібліотеки валідації
(yup, Formik).
Приклад: форма пошуку роботи на сайті вакансій із перевіркою введених даних.
5. Міні-динаміка через JavaScript
Додати динамічний функціонал на сайт:
o слайдер для зображень (Swiper.js, Flickity),
o динамічний список завдань,
o лічильник кліків або таймер.
Приклад: в системі управління завданнями реалізувати drag & drop для перетягування
елементів.
7. Додавання інтерактивних компонентів Figma
Використати можливості Figma для створення інтерактивних прототипів.
Додати анімації та ефекти переходів у макеті.
Приклад: у дизайні сайту школи зробити інтерактивне меню.
9. Додавання карт із інтерактивними точками
Використати Leaflet.js або Google Maps API для інтерактивної карти.
Додати точки з інформацією або маршрути.
Приклад: для медичного порталу показувати розташування клінік на карті.
10. Підключення API для отримання даних
Використати відкриті API для завантаження контенту (Weather API, News API).
Використати fetch() або Axios для отримання даних.
Приклад: на сайті погоди виводити прогноз із реального API.
11. Впровадження WebSockets або AJAX
Використати WebSockets (Socket.io) або AJAX-запити (XMLHttpRequest, fetch).
Приклад: для форуму або чату реалізувати обмін повідомленнями в реальному часі.
12. Онлайн-запис на події чи прийом
Реалізувати інтерактивний календар з можливістю бронювання подій або запису на
консультацію.
Використати localStorage або IndexedDB для збереження записів.
Приклад: на сайті клініки реалізувати систему онлайн-запису до лікаря.
13. Збереження користувацьких налаштувань у браузері
Реалізувати можливість збереження теми, обраного міста, мови сайту.
Використати localStorage, sessionStorage.
Приклад: на агрегаторі подій зберігати обрану категорію заходів.
14. Додавання push-сповіщень
Реалізувати push-сповіщення у браузері через Notification API.
Приклад: на сайті новин повідомляти про нові статті.
 15. Рейтингова система
Додати можливість залишати оцінки та коментарі.
Використати stars-rating через CSS або JavaScript.
Приклад: у системі відгуків зробити можливість ставити рейтинг товару.
16. Реалізація чат-бота
Додати інтерактивного чат-бота, який допомагатиме користувачу.
Використати Dialogflow або просту логіку на JavaScript.
Приклад: у системі пошуку роботи зробити бота, який пропонує вакансії за ключовими
словами.
